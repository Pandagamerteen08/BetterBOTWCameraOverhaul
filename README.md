# 1. Project Identity

## Better Breath of the Wild (BOTW) Camera Overhaul

**Better Breath of the Wild (BOTW) Camera Overhaul** is a complete camera enhancement framework for **The Legend of Zelda: Breath of the Wild** designed to improve, expand, and modernize the game's camera system while preserving the original design philosophy created by Nintendo.

This project is not designed to replace Breath of the Wild's identity.

Nintendo created one of the most carefully designed third-person cameras in modern gaming, balancing:

- Exploration
- Combat
- Traversal
- Environmental awareness
- Cinematic presentation

Instead, Better BOTW Camera Overhaul expands that foundation by giving players complete control over their camera experience.

The goal is simple:

> Keep the feeling of Breath of the Wild while removing the limitations of the original camera system.

---

# Project Mission Statement

Better BOTW Camera Overhaul exists to provide:

- Complete camera customization
- Improved accessibility
- Better cinematic control
- Advanced photography tools
- Creator-focused camera systems
- A flexible camera framework for the Breath of the Wild modding community

Players should be able to experience Hyrule from their preferred perspective.

Whether that means:

- A traditional Nintendo-style camera
- A cinematic exploration camera

- A content creator camera
- A competitive combat camera

the choice belongs to the player.

# 19. Developer Documentation

This section is intended for developers, contributors, advanced users, and anyone interested in understanding how Better Breath of the Wild (BOTW) Camera Overhaul works internally.

The project is designed as a modular camera framework instead of a collection of individual camera edits.

The goal is to create a flexible, expandable, and maintainable camera system.

---

# Development Philosophy

Better BOTW Camera Overhaul follows several core principles:

## Modular Design

Each major system is separated.

Examples:

- Camera Core
- Camera States
- Configuration
- Profiles
- User Interface
- Preview System
- Creator Tools

This allows individual systems to be improved without rewriting the entire project.

---

## User Customization First

Almost every camera behavior should be configurable.

Developers should avoid creating hardcoded camera values whenever possible.

Instead:

Bad:

CameraDistance = 8

Preferred:

CameraDistance = UserSetting.CameraDistance

---

## Preserve Vanilla Behavior

The default configuration should always respect Breath of the Wild's original camera design.

New features should enhance the experience rather than completely replace it.
